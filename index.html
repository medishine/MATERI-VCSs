Version Control System

Sistem yang mengelola perubahan dari sebuah dokumen program komputer, website dan kumpulan informasi lain. Karena sistem ini merekam perubahan yang terjadi pada sebuah dokumen atau sekumpulan dokumen dari waktu ke waktu maka dimungkinkan untuk kembali ke versi tertentu di kemudian hari.

Jenis-Jenis Version Control System

Local Version Control System

Metode version-control yang digunakan oleh banyak orang adalah menyalin dokumen ke folder lain. Local VCSs yang memiliki sebuah database sederhana yang menyimpan seluruh perubahan terhadap dokumen dalam revision control.

Centralized Version Control System

Sistem ini memiliki sebuah server tunggal yang berisi seluruh versi dari dokumen, dan sejumlah orang yang mengunduh dokumen dari tersebut. Kelebihan: semua orang tahu hingga batas tertentu apa yang orang lain lakukan dalam proyek. Kekurangan: Jika server mati maka tidak bisa berkolaborasi.

Distributed Version Control System (DVCS)

Dalam DVCS (seperti Git, Mercurial, atau Darcs), klien tidak hanya mengunduh salinan terbaru dari dokumen; melainkan, menyalin seluruh repositori, termasuk riwayat lengkapnya.

Git

Git adalah distributed version control software system yang mampu menangani perubahan dari source code atau data.

1. modified artinya berkas telah diubah namun belum men-commit-nya ke database.
2. staging artinya file yang diubah pada versi saat ini telah ditandai untuk direkam pada commit selanjutnya.
3. committed artinya data telah secara aman disimpan di lokal database.

Github

Github adalah sebuah platform yang memungkinkan developer untuk membuat, menyimpan, mengatur serta membagikan kode yang dibuat. Github menggunakan git untuk menyediakan distributed version control dan github sendiri menyediakan kontrol akses, tracking bug, software feature request, task management, continues integration serta wikis untuk setiap proyeknya.

Perintah Dasar pada Git

Git Init

Berfungsi untuk membuat repository kosong yang baru atau mengubah folder yang telah ada menjadi sebuah git repository.

Git Add

Berfungsi untuk menambahkan dan mendata perubahan yang terjadi ke staging area, perubahan-perubahan yang sudah di-add inilah yang nantinya direkam pada saat commit dilakukan.

Git Restore

Git restore digunakan untuk menghapus perubahan yang telah dilakukan. git restore .

Git Commit

Berfungsi untuk menyimpan seluruh perubahan yang ada pada staging area.

Git Log

Digunakan untuk menampilkan daftar commit yang telah dilakukan pada repository.

Git Checkout

Berfungsi untuk beralih ke branch atau commit tertentu. Untuk beralih ke branch tertentu gunakan perintah. Untuk beralih ke commit tertentu gunakan perintah.

Git Branch

Perintah untuk melihat branch yang ada pada repository. Untuk membuat branch baru dapat menggunakan perintah.

Git Merge

Digunakan untuk menggabungkan branch, dapat dilakukan dengan perintah.

Git Push

Berfungsi untuk mengirim perubahan berupa commit yang dilakukan ke remote repository.

Git Pull

Fungsinya adalah untuk mengambil perubahan dari remote repository dan biasanya digunakan untuk menggabungkan perubahan dari remote repository ke dalam repository lokal.

Git Fetch

Perintah ini hanya mengambil perubahan dari remote repository tanpa menggabungkannya.

Git Clone

Berfungsi untuk menyalin remote repository milik pribadi maupun milik orang lain (selama repository yang dimaksud public) ke komputer yang menjalankan perintah tersebut dan akan tersimpan secara lokal.
